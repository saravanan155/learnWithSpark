# Learn with Spark — Build Plan

> Our single reference for what we're building and in what order. We build in **small batches:
> one concept per commit**, each readable in a couple of minutes and tested before moving on.

## What this is

A **Week 3 Agentic AI** project (The Gen Academy, "Mastering Agentic AI"). *Learn with Spark* is
an AI-literacy game for kids: the learner teaches **Spark**, a robot with an "empty brain." The
engineering substance is that **game levels are generated by a multi-agent LangGraph pipeline**.

## Scope right now (deliberately narrow)

- **Backend only — no frontend.** Everything stays in Python where the agentic concepts live.
- The "game" part **stops at the coding agent producing the game code as text** and the
  **testing agent producing a report** on it. We do not render or play anything yet.
- Once the agentic workflow is solid and understood, we add functionality + a frontend in stages.

## Requirements this project must demonstrate (from the Week 3 handout)

**Course constraints**
- Build track: **LangChain + LangGraph (Python)**.
- **At least one model call via Nebius Token Factory.**
- Deliverables: **GitHub repo** (+ architecture diagram), **demo video ≤ 5 min**, **documentation**.

**Agentic concepts to prove**
| Concept | "Proving it" looks like |
|---|---|
| Multi-agent | 2+ specialized agents, each one job |
| Control flow | the graph decides what step comes next |
| State | data held + passed across steps |
| Tool usage | agents call tools; mark **read** vs **write** |
| Checkpoint / persistence | state survives steps *and a restart* |
| Human-in-the-loop | pauses for approval on write actions |
| Error / tool-failure recovery | a tool fails → retry / ask / stop, not crash |

Handout warning: *"If your agent works on the happy path but falls over on the first tool
failure, you have not finished."* → error handling + HITL are where the grade is.

## The batch plan

### Phase A — Learn the LangGraph machinery (stub logic, no LLM, no API keys)
| # | Batch | Concept it teaches |
|---|---|---|
| B1 | Repo scaffold + this PLAN.md | project setup |
| B2 | A `State` + one `research` node (fake ideas) + a runner | state + node + running a graph |
| B3 | Add a `guardrail` node + an edge | data flowing between nodes |
| B4 | Add the SqliteSaver checkpointer | persistence (survives a restart) |
| B5 | Human pause after research ("pick an idea") | human-in-the-loop (interrupt/resume) |
| B6 | A conditional route (guardrail: safe → finish, unsafe → blocked) | control flow (graph decides) |

### Phase B — Make the agents real (needs Nebius + Anthropic keys)
| # | Batch | What becomes real / concept |
|---|---|---|
| B7 | Wire Nebius; `research` calls a real model | model usage + Nebius requirement |
| B8 | `guardrail` calls a real model + its approval gate | 2nd agent + 2nd gate |
| B8.5 | Research gate gets real teeth: **accept / edit / regenerate / abandon** (regenerate capped at 3) | richer HITL — the human shapes, rejects-and-regenerates, or cleanly abandons ideas; the graph loops back to `research` or stops |
| B9 | `coding` agent (Claude) — generates game code as text | multi-agent |
| B10 | `testing` agent — checks the code, writes a report | the test step |
| B11 | Repair loop + escalation (test fails → fix, max 3) | error/tool-failure recovery |
| B12 | `research` gets a real read tool (web/mock), ReAct-style | tool usage (read) |
| B13 | `publish` writes to SQLite — gated behind a human | tool usage (write) + writes-deserve-a-human |
| B14 | Harden tool-failure handling everywhere | robustness |

### Phase C — Later, in stages (only after B is solid)
Revisit/edit + versioning · frontend (admin + learner UI) · Sandpack to play levels ·
LangSmith tracing · MCP server.

## Tech decisions
- **LangGraph (Python)** for orchestration; **SqliteSaver** checkpointer for persistence.
- **Nebius Token Factory** for research / guardrail / judge (cheap + course-required).
- **Claude** for the coding agent (hardest reasoning).
- **SQLite** for the published-level store (added at B13).
- Package manager: **uv**, Python **3.12**.
